# Security Policy

## Supported Versions

We provide security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| < 1.1.0 | :x:                |

## Reporting a Vulnerability

The Polarys Foundation team takes the security of PolarysDB seriously. If you believe you have found a security vulnerability in PolarysDB, please report it to us as soon as possible.

**Please do not report security vulnerabilities via public GitHub issues.**

### Reporting Process

1.  **Email us:** Send a detailed report to [EMAIL_ADDRESS]
2.  **Include details:** Provide a clear description of the vulnerability, steps to reproduce it, and any potential impact.
3.  **Acknowledgment:** We will acknowledge receipt of your report within 48 hours.
4.  **Investigation:** Our team will investigate the issue and may contact you for further information.
5.  **Resolution:** We will work on a fix and keep you updated on the progress.
6.  **Disclosure:** Once a fix is available and tested, we will release a new version and publicly disclose the vulnerability (with credit to you, if desired).

## Disclosure Policy

We follow a coordinated disclosure process to ensure that users have time to update their systems before the details of a vulnerability are made public. We ask that you give us a reasonable amount of time to address the issue before disclosing it to others.

## Recognition

We value the contributions of security researchers and will happily credit you for your discovery in our security advisories, unless you prefer to remain anonymous.
